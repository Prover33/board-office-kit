# config.example.md · 配置区填写示范（虚构演示）

> ⚠️ **本文件仅作格式示范**，不是专家读取的配置。把值**抄进** `agents/board-office.md` 的【配置区】即可。
> 示例公司为虚构，请勿当作真实数据。

---

## 必填（2 项——不填这两项，专家无法指代你的公司）

| 配置项 | 示例填写 | 说明 |
|---|---|---|
| 公司全称 | `示例智造股份有限公司` | 目标上市公司全称 |
| 证券代码 | `SH/609000` | 交易所 + 代码 |

## 选填（6 项——每多填一项，某方面能力升一级）

| 配置项 | 示例填写 | 不填时降级为 |
|---|---|---|
| 上市板块 | `主板` | 通用 A 股全板块规则 |
| 是否国资控股 | `否` | 按非国资处理（若你实际是国资，务必填"是"） |
| 国资审批链 | `不适用（非国资）` | 国资路径不生效 |
| 治理架构 | `单层制（董事会+审计委员会，不设监事会）` | 默认单层制 |
| IMA 知识库 ID | `（填你的库ID）` | 制度问答走公开法规+通用口径 |
| 公文模板目录 | `（留空或填本地路径）` | 起草公文给通用骨架 |

## 占位符替换

抄完上表后，把 `board-office.md` 正文里的 `<占位符>` 也对应替换：

- `<TARGET_CO>` → `示例智造股份有限公司`
- `<TICKER>` → `SH/609000`
- 其余 `<...>` 按上表类推

替换后角色定位段变为：

> 你是 **示例智造股份有限公司**（证券代码 **SH/609000**，**主板**）董事会办公室的资深证券事务专家……

## 零配置起步

暂时一项都不想填？也完全可以直接用。专家会以"通用 A 股上市公司董办"身份干活，用"贵公司"指代你。等方便时再填，每多填一项就更精准一分。
