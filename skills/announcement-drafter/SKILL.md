# 海南华铁临时公告起草技能

## 定位

把"写一份公告"从每次重新交代，变成一条确定的流水线：**识别类型 → 扫历史同类 → 套母版/参考同业 → 写初稿 → 自检交付**。

> 适用范围：**交易所临时公告**（董事会决议、股东大会决议、对外担保进展、减持计划、回购方案、权益分派、章程变更、换届、业绩说明会等）。
> 不适用：**议案类 / 一般公文**（党委会议案、建议书、函）——那类用 `00-一般公文格式规范.md`，标题字体是黑体/楷体/仿宋，**与本公告规范不同**（见下文"坑"）。

## 一、样式速览（实测 DNA，详见 `references/style_spec.md`）

| 要素 | 规范（样式 ID / 母版实现） |
| --- | --- |
| 页面 | A4，页边距 上/下 2.54cm、左/右 3.18cm |
| 公告头（代码行） | 样式 `af4` 居中；**真名版预填** `证券代码：603300  证券简称：海南华铁  公告编号：2026-XXX`，**脱敏版留空值** |
| 公告标题区 | 样式 `af5` = 黑体/小二(18pt)/加粗，且**红色（run 级 color EE0000）**；公司全称行居中、公告名称行左对齐首行缩进 2 字符 |
| 免责声明 | **红色 + 黑体18pt + 文本框(方框)**，固定 boilerplate，由母版文本框承载（脚本保留、不重建） |
| 正文 | 样式 `afa` 宋体小四、1.5 倍行距、首行缩进 2 字符、两端对齐 |
| 标题一/二/三级 | **统一样式 `af7`**（宋体/加粗/顶格）——新母版里 `（一）`/`1.` 也加粗；三级编号 `1.文字` 数字后**无空格** |
| 重要内容提示 | 样式 `afb`（加粗），后接 `afa` 正文段 |
| 落款 | 样式 `afc` 右对齐；真名版 `浙江海控…董事会`，脱敏版 `XX公司董事会` |

**⚠️ 公告标题字体铁律**：层级标题用**宋体+加粗**区分，不是黑体/楷体（那是议案类的规矩）。别混。

**⚠️ 公司全称**：`浙江海控南科华铁数智科技股份有限公司`（带"浙江"前缀，证券简称"海南华铁"，代码 603300）。**不是"华铁应急"**（那是曾用名）。

## 二、写作工作流（SOP）

### 第 1 步 · 识别类型
判断公告属于哪一类（决议类 / 进展提示类 / 权益分派类 / 减持回购类 / 章程变更类等），确定落款机构（董事会 / 监事会 / 股东大会）。

### 第 2 步 · 扫描历史同类
用技能脚本 `scripts/find_similar.py` 在 Hermes 文件名索引里按关键词筛同类公告（按日期降序），或直接在 `D:\File\02-公告\01-公告发布\` 按年份+关键词找：
```bash
PYTHON="C:/Users/admin/.workbuddy/binaries/python/envs/default/Scripts/python.exe"
SKILL="C:/Users/admin/.workbuddy/skills/announcement-drafter"
"$PYTHON" "$SKILL/scripts/find_similar.py" "担保" --limit 10
"$PYTHON" "$SKILL/scripts/find_similar.py" "减持" --year 2025
```
命中后**优先取最近一份**作为结构参考。

### 第 3 步 · 分支
- **有同类** → 打开最近一份，套 `04-公告通用母版.docx`（真名版），只替换：日期、金额、议案、表决结果、落款机构；结构基本照搬。用 `scripts/new_announcement.py` 生成时传入对应 JSON 即可。
- **无同类** → 参考其他上市公司同类公告（巨潮网 cninfo 全文检索，可调用 `announcement-case-analyzer` 技能的 `search_cninfo.py` 搜"××公告 范本"；或用 WebSearch）+ **向用户询问基础信息**（代码、日期、核心数据、适用法规、审议程序）→ 基于母版写初稿。

### 第 4 步 · 生成文档
用 `scripts/new_announcement.py` 数据驱动生成（详见脚本头部注释）。脚本以 04/05 母版为基底，**固定字段原地替换 run 文本、可变区（重要提示+章节）按新样式 ID（af4/afa/af5/af7/afb/afc）重建**，全程保留母版免责声明文本框：
```bash
"$PYTHON" "$SKILL/scripts/new_announcement.py" \
  --base real \                         # real=真名版(04) / mask=脱敏版(05)
  --json /path/to/announcement.json \
  --out "D:/File/02-公告/01-公告发布/2026/XXX-...公告.docx"
```
- 脱敏版（`--base mask`）会自动把公司全称/落款强制替换为 `XX公司`、代码行留空，**保证真名零残留**，用于对外演示/培训。
- 不需要的占位段（如"重要内容提示"）在 JSON 里不写即可。

### 第 5 步 · 过自检清单（见 `references/style_spec.md` 第四节）后交付
用 `present_files` 给出 docx，并在对话中说明：套了哪份历史公告 / 参考了什么 / 哪些数据待用户确认。

## 四、坑与注意（必读）

- **❌ Word 字体回退（最致命）**：生成 docx 时，`w:ascii`/`w:hAnsi` 必须填**西文字体**（Times New Roman），`w:eastAsia` 才填中文（宋体/黑体）。若把中文名塞进 `w:ascii`，Word/WPS 会**整段回退成等线**——表现为"格式全乱"。本技能脚本已按此规范实现，**禁止绕过脚本手填字体名到 ascii 属性**。
- **✅ 以真实公告 docx 为基底**：不要用 python-docx 空白模板自建（theme/字体表简化、兼容性差）。`new_announcement.py` 以 `04/05` 母版为基底，样式已锁死，只增删段落。
- **⚠️ 免责声明是文本框，python-docx 会把它 flatten（丢 drawing）**：故 `new_announcement.py` 改用 **lxml 直接改 `word/document.xml`**，只替换固定字段 run 文本、清空可变区后按新样式 ID 重建段落，绝不触碰免责声明文本框，完美保留红字方框。
- **新母版用自动样式 ID（af4/afa/af5/af7/afb/afc），不是命名样式**：脚本按这些 ID 引用；标题区（af5）为红色黑体，红色在 run 级；一/二/三级标题统一 af7（均加粗）。
- **公告标题两行的对齐不同**：公司全称行居中、公告名称行左对齐缩进 2 字符（不能居中叠缩进）。脚本已分别处理。
- **三级标题编号 `1.文字` 无空格**，不是 `1. 文字`。
- **公司全称带"浙江"前缀**，曾用名"华铁应急"已弃用（2024-10 更名）。
- **"公告"≠"结尾带公告二字"**：公司章程、治理准则等上网文件也属披露文件，但本技能只管"临时公告"正文，不写议案/函。
- 巨潮网 cninfo 全文检索 API 仅索引标题+摘要、不索引 PDF 正文；搜同业范本主要靠标题命中 + 用搜索引擎抠正文。
- 法规判断仅供参考，重大合规决策结合最新法规与律师意见。
