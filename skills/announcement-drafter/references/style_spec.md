# 临时公告样式规范（技能内副本）

> 提炼自 `D:\Dx\08-Prvatie\Hermes\3-董办\08-模板\03-公告格式规范与写作工作流.md`，供 announcement-drafter 技能离线参考。
> 最后同步：2026-07-21（对齐用户当日手改的 `04-公告通用母版.docx`）。

## 一、整体样式硬规范

| 要素 | 规范（样式 ID / 母版实现） |
| --- | --- |
| 页面 | A4，页边距 上/下 2.54cm、左/右 3.18cm |
| 公告头（代码行） | 样式 `af4` **左对齐**；真名版预填 `证券代码：603300  证券简称：海南华铁  公告编号：2026-XXX`，脱敏版留空值 |
| 公告标题区 | 样式 `af5` = 黑体/小二(18pt)/加粗，且**红色（run 级 color EE0000）**；公司全称行与公告名称行**均居中、无缩进** |
| 免责声明 | **红色 + 黑体18pt + 文本框(方框)**，固定 boilerplate，母版文本框承载（生成时保留、不重建） |
| 正文 | 样式 `afa` 宋体小四、1.5 倍行距、**首行不缩进（顶格）**、两端对齐 |
| 落款 | 样式 `afc` 右对齐（真名版`浙江海控…董事会`／脱敏版`XX公司董事会`） |
| 重要内容提示 | 样式 `afb`（加粗）+ `afa` 正文段 |

## 二、标题层级规范（一/二/三级）

**核心规律**：公告内的层级标题用**与正文相同的宋体**，靠**加粗 + 中文编号**区分（不是黑体/楷体）。

| 层级 | 编号 | 字体 | 字号 | 加粗 | 对齐/缩进 |
| --- | --- | --- | --- | --- | --- |
| 一级 | `一、` | 宋体(af7) | 小四 | 是 | 首行缩进 2 字符（firstLineChars=200） |
| 二级 | `（一）` | 宋体(af7) | 小四 | 是 | 首行缩进 2 字符（firstLineChars=200） |
| 三级 | `1.文字` | 宋体(af7) | 小四 | **是（新母版 af7 统一加粗）** | 首行缩进 2 字符（firstLineChars=200）（数字后**无空格**） |

> 注：新母版（用户在 Word 手改）把一/二/三级标题**统一用 `af7` 样式（均加粗、首行缩进 2 字符）**，即 `（一）`/`1.` 也加粗；与早期"三级常规"规范不同，**以当前母版为准**。

## 三、与"议案类 / 一般公文"的区别

| 元素 | 议案类（党委会议案等） | 本公告规范（临时公告） |
| --- | --- | --- |
| 一级标题 | 黑体 | 宋体+加粗 |
| 二级标题 | 楷体 | 宋体+加粗 |
| 三级标题 | 仿宋+加粗 | 宋体（常规） |
| 大标题 | 宋体二号居中 | 黑体小二居中 |
| 落款 | 右对齐 | 右对齐（一致） |

判断口诀：交易所披露的"××公告"→ 用本公告规范（宋体标题）；上董事会/股东会的"关于××的议案"、给监管的"建议书/函"→ 用议案类规范。

## 四、自检清单（写完必过）

- [ ] 公司全称 = 浙江海控南科华铁数智科技股份有限公司（非华铁应急），标题区红色黑体
- [ ] 证券代码行（af4）**左对齐**、代码/简称/编号齐全（真名版预填 603300/海南华铁；脱敏版留空）
- [ ] 公告标题：公司全称行与公告名称行均黑体小二加粗居中（红色）、无缩进
- [ ] 免责声明红字黑体 + 文本框方框、措辞完整
- [ ] 正文（afa）宋体小四、1.5 倍行距、首行不缩进（顶格）
- [ ] 一/二/三级标题统一 af7 加粗、**首行缩进 2 字符（firstLineChars=200）**；三级编号 `1.文字` 无空格
- [ ] 落款机构（afc）与类型一致、右对齐、日期正确
- [ ] 数字与中文单位间无空格（`2026年7月20日`）；引用文件用书名号《》

## 五、实现要点（生成母版/公告的技术注意）

- **字体必须按 Word 规范拆分**：`w:ascii`/`w:hAnsi` 填西文（Times New Roman），`w:eastAsia` 才填中文（宋体/黑体）。中文名塞进 `w:ascii` 会让 Word/WPS 整段回退成等线。
- **以真实公告 docx 为基底**：别用 python-docx 空白模板自建，真实文件 theme/字体表原生导出、渲染一致。
- **缩进/对齐由母版驱动**：`new_announcement.py` 的 `make_para()` 重建段落时整段复制母版对应样式代表段落的 `ind`+`jc`，故母版任何缩进/对齐调整都会自动继承——改母版后无需改脚本，但本规格文档须同步。
