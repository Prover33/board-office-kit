# 上市公司公告案例分析技能

## 定位
把"某类上市公司公告"变成一份**可归档、可汇报**的案例分析报告：检索 → **先一段话总览** → 再逐案分析 → 输出带原文链接的 Word。
搜索逻辑参考万得(Wind)公告检索五步法，已适配巨潮网(cninfo)。

## 能力边界一览表（使用前必看）
> 评测反馈：先把"能做/不能做"说清楚，避免用户白费时间。

| 能力 | ✅ 支持 | ❌ 不支持 / 需手动 |
|---|---|---|
| 搜索范围 | 沪深北 A 股上市公司公告（标题+摘要） | 港股(HKEX)、美股、新三板精选层以外 |
| 搜索方式 | 关键词、公司代码、全文检索、同义词扩展 | PDF 正文内容搜索（API 不索引正文） |
| 行业 / 板块筛选 | 可按关键词 + 板块词粗筛（如 `白酒 分红`、`创业板 一季报`） | 不支持申万 / 中上协行业**直接过滤**（cninfo API 无行业参数）；需某行业全覆盖时，先建该行业股票清单再逐家 AND 关键词检索 |
| 公司章程条款 | 先搜到章程文件清单 → 再下载 PDF 提取 / 或用搜索引擎抠条款 | 无法通过 API 直接搜到章程某条具体写法 |
| 数据源 | 巨潮网 cninfo（免费） | 易董需用户手动导出粘贴；Wind 无接口 |
| 报告输出 | Word 三色块报告（结论速览+法律分析+局限性+逐案分析） | PPT / Excel / 在线文档 |
| 自动化程度 | 全自动检索→筛选→生成报告，人工确认即可 | 章程正文提取需额外一步下载 |
| 出错恢复 | 脚本内置自动重试（最多3次指数退避）；失败率>30%时打印警告提示人工判断 | 极端情况全部失败后才需手动干预 |

## 两种数据模式
- **自动模式（巨潮网 cninfo 全文检索，首选）**：用 `scripts/search_cninfo.py` 调用巨潮网全文检索接口，按关键词 / 公司代码 / 全文搜全市场 A 股公告，返回结构化清单（标题、代码、日期、稳定的 `detailUrl` 原文链接）。免费、覆盖沪深北、支持全文 + 关键词。
- **易董补录模式**：用户粘贴**易董（价值在线）**筛出的案例文本 / 标题 / 链接。注意：易董无自动连接器、也无法代登录，案例须用户登录易董后导出或复制进来；本技能用与自动模式完全相同的模板和排版加工。

## 触发条件（何时启用本技能）
用户提问命中以下任一场景即触发：
- 明确要求查公告："帮我找 XX 的年报 / 资产重组公告"
- 涉及公司事件 / 行为："XX 最近有没有分红 / 高管变动情况"
- 需要官方文件佐证："XX 并购案的官方披露 / 退市原因"
- 政策 / 监管合规查询："最新的减持新规 / 某行业监管政策"
- 泛指案例研究："搜几个 XX 类型的上市公司案例出报告"

### ❌ 以下场景**不适用**本技能（请改用其他工具/技能）
| 你的需求 | 为什么不适合 | 建议用什么 |
|---|---|---|
| "帮我写一份制度 / 起草章程" | 本技能只**检索已有公告**，不生成新文档 | 用文档生成类技能（如 docx/pptx 技能） |
| "查一下这只股票现在的行情 / 价格" | cninfo 是**公告披露平台**，不是行情数据源 | 用金融数据类技能（westock/westock-tool/neodata） |
| "帮我看一下这份合同有没有法律风险" | 本技能检索的是**上市公司公开公告**，不是合同审查 | 用法律专业工具或律师审核 |
| "搜一下学术论文 / 行业研报" | 覆盖范围仅限 **A股上市公司法定披露文件** | 用学术搜索 / 研报数据库 |
| "帮我整理桌面上的这些文件" | 本技能是**信息检索+报告生成**，不做文件整理 | 用通用文件处理能力 |
| "港股 / 美股公司有 XX 公告吗" | cninfo 仅覆盖 **沪深北 A 股**，不含港股/美股 | 用对应市场数据源（港交所披露易 / SEC EDGAR） |
| "实时监控某公司的最新公告" | 本技能是**按需手动触发**的，无后台定时监控功能 | 用自动化任务（automation）+ 定时调用脚本实现 |

## 工作流程（核心：Wind 五步法适配 cninfo）
> 原则：**先精确化"实体 + 时间"，再用精准查询检索，最后人工筛选去噪**。不要一上来就关键词堆砌碰运气。

### Step 1 · 实体识别与标准化
把用户口中的模糊主体转成精确 6 位代码，确保后续搜索用的是代码而非昵称：
- 查 `references/entity_code_map.md` 的昵称→代码表（已内置 海南华铁 603300 等董办关注标的）。
- 命中即取代码；未命中则 `WebSearch "<公司名> 股票代码"` 解析，必要时列出多个候选让用户确认。
- 人名 / 产品名同理标准化（如"茅台"→600519）。

### Step 2 · 时间范围确定
按用户表述解析搜索窗口（cninfo 接口本身不带日期参数，本步用于给 `search_cninfo.py` 传 `--start/--end` 后置过滤）：
- 明确年份 / 季度：直接用（如"2024 年报"→2024-01-01 ~ 2025-04-30）。
- 相对时间（"最近 / 近期"）：默认近 1–3 个月；无时间限定默认近 6 个月。
- 节假日 / 交易日等模糊表述：合理推断一个区间。

### Step 3 · 查询构建与执行
> ⭐ **本步骤最核心，直接决定搜得准不准。** 下面用"普通语言→怎么操作"对照着写。

把用户自然语言意图转成**精准检索式**再执行（不是关键词堆砌）：

#### 🔑 三种核心检索方式（按精准度排序）
| 方式 | 写法示例 | 适用场景 | 为什么准 |
|---|---|---|---|
| **代码+行为**（最推荐） | `603300 回购` `603300 行政处罚` | 已知具体公司 | API 服务端按公司代码收窄结果，比纯关键词精确得多 |
| **纯行为/全市场** | `违规担保` `业绩承诺` | 不限定公司，看全市场 | 靠关键词 + `--type` 过滤 |
| **精确短语** | `"控股权转让协议"` | 防止拆词误匹配 | 引号内必须完整出现才算命中 |

> 💡 **术语解释**："布尔检索"就是支持 AND/OR/NOT 逻辑组合；本技能已封装好，你只需写中文关键词即可。

#### 📌 同义词扩展（默认开启，提升召回率）
- 开关：默认 `--expand` 开启；关闭用 `--no-expand`
- 原理：输入 `并购` → 自动同时搜 [并购, 收购, 受让, 购买资产, 股权收购] → 按 `announcementId` 融合去重
- 效果：显著减少"换了个说法就搜不到"的情况
- 规则详见 `references/query_decomposition.md`

#### 📌 多公司/多类型：并行不串行
同时查多个公司或多种行为时，**多次调用脚本并行跑**，不要一个等完再跑下一个。

#### 命令示例
PYTHON="C:/Users/admin/.workbuddy/binaries/python/envs/default/Scripts/python.exe"
SKILL="C:/Users/admin/.workbuddy/skills/announcement-case-analyzer"
# 例1：某公司某类行为（代码内嵌，服务端收窄）
"$PYTHON" "$SKILL/scripts/search_cninfo.py" "603300 行政处罚" --pagesize 20 --json /tmp/cninfo.json
# 例2：全市场某类行为 + 时间窗 + 类型词过滤（自动翻页覆盖非近期匹配）
"$PYTHON" "$SKILL/scripts/search_cninfo.py" "违规担保" --start 2025-07-15 --end 2026-07-15 --type 担保 --pagesize 15 --json /tmp/cninfo.json
```
- 过滤参数：`--code`（按 secCode 校验/收窄）、`--start/--end`（公告日期窗）、`--type`（按**标题**含某词过滤，因 `announcementType` 是数字代码如 `01010503`）。
- 若 API 不可达，退回 `references/search_cookbook.md` 的 WebSearch 模板（`site:cninfo.com.cn "<关键词>"`）。

### 章程 / 上网文件检索（进正文抠条款，不是只搜公告标题）
用户要"公司章程里怎么写 XX"时，属于**上网文件**检索——公司章程、董事会秘书工作制度、总经理工作细则等治理文件在巨潮网/交易所归档为独立披露文件（标题多为"XX公司章程（YYYY修订）"），**结尾不是"公告"二字也是披露文件，务必纳入检索范围**。要点：
- cninfo 全文检索 API **只索引标题+摘要、不索引 PDF 正文**（已实测：`陈述申辩`仅 4 条、`行政处罚事先告知书` 1041 条）。故 `search_cninfo.py "公司章程"` 能按标题捞出章程文件清单（全市场 14.7 万份），但**拿不到正文条款**——别误以为"搜不到"就不存在。
- 章程正文两种取法（优先 b，最快）：
  - (a) 用 `search_cninfo.py` 按标题捞出章程文件链接清单，再对每篇下载 PDF + 文字提取工具（PyMuPDF / fitz）提取正文（`pip install pymupdf`）。
  - (b) **搜索引擎（已爬 PDF 全文）直接用条款原文短语 + `公司章程` 检索**，返回含条款摘录与原文链接的结果（如 `"董事会秘书由董事长提名" 公司章程`）。这是抠章程条款最快的路。
- 💡 **术语解释**：`detailUrl` = 巨潮网的公告详情页链接；这个页面是 JS 动态渲染的，自动抓取工具往往只能拿到标题。要拿到正文内容，需要找到真实的 PDF 或 HTML5 版本（搜索引擎结果里常有）。
- 区分"公司章程"与"董事会秘书工作制度/细则"：制度系"依据《公司章程》制定"，其董秘提名等条款与章程一致，可作章程约定旁证；报告须注明每条是"公司章程"原文还是"工作制度（引章程）"。
- 上位规则（抠条款时一并核对）：证监会《上市公司董事会秘书监管规则》(公告〔2026〕8号，2026-05-24 施行，第二十三条要求董事会提名委员会对董秘人选遴选、审核)、《上市公司治理准则(2025年10月修订)》第四十七条、《上市公司章程指引》。

### Step 4 · 结果筛选与相关性排序
`search_cninfo.py` 已按"标题命中关键词 → 日期降序"排序，仍需二次筛选去噪：
- **时间匹配**：公告日期是否在 Step 2 窗口内。
- **类型匹配**：年报 / 季报 / 临时公告 / 监管函是否对应用户需求。
- **实体匹配**：文档公司是否与目标公司一致。
- **内容相关性**：标题 / 摘要是否真回答了问题。
- 只保留直接相关文档，不堆砌无关结果；每篇用真实 `detailUrl` 作为原文链接。

#### ⭐ 系列公告合并与事件归类（出报告前必做，借鉴易董公告专家）
cninfo 一次检索会返回同一事项的**多份程序性公告**（如股东会通知+决议+法律意见书），直接"逐条=逐案"会让报告很水。出报告前先做两步：

**1. 系列公告合并为一条事件**：
- 合并组合：股东会通知+决议+法律意见书 → **股东会**；减持计划+进展+完成/触及1% → **减持**；同一股东同月多次质押/解质押 → **股权变动**；业绩说明会提示性公告+纪要 → **业绩说明会**；分拆上市预案+保荐核查+法律意见+董事会决议+独董意见+股东会通知 → **分拆上市**。
- 规则：程序性/配套性公告（单独的法律意见书、董事会决议、独董意见）能并入主事项就并入；**不同独立事项不得合并**（股权激励回购注销、关联交易、分红相互独立，必须分别列出）。
- 合并后日期只填一个：优先主公告或最终进展公告披露日；正文链接可并列多个。

**2. 每条事件归类到固定事件类型**（统一标签，便于统计与横向对照）：
重大合同 / 对外投资 / 资产交易 / 诉讼仲裁 / 担保 / 股权变动 / 减持 / 增持 / 回购 / 股权激励 / 定增 / 可转债 / 分红 / 关联交易 / 分拆上市 / 股东会 / 董监高变动 / 业绩说明会 / 其他——共 19 类，**每条只选一个**；涉及多类则拆成多条独立事件。报告 `cases[].type` 从该枚举取值（见 `references/report_template.md`）。

#### ⚠️ 异常处理速查（出错时按此表排查）
| 现象 | 可能原因 | 解决方案 |
|---|---|---|
| 报错 `请求失败` / 超时 | 巨潮 API 临时不可达或网络抖动 | **脚本已自动重试**（最多3次，3s→6s→12s退避）；若控制台出现 `[fail] 已重试3次均失败` 才需人工介入——先检查网络再重跑；仍不行则切换 WebSearch 兜底（见下） |
| 搜到结果为 0 | 关键词太窄 / 拼写错误 / 该主题确实无公告 | 先确认关键词没错 → 去掉 `--type` → 放宽 `--start/--end` → 开启 `--expand`；仍为 0 则如实告知用户 |
| 结果数量明显偏少（平时能搜 50+ 这次只有几个）| **网络波动导致部分请求失败**（最常见） | 脚本控制台会打印 `[提示] 结果可能不完整`（>30%请求失败时）；**看到此提示应重新跑一次命令**，不要直接用不全的结果出报告 |
| JSON 解析错误 | API 返回格式变化（极少见） | 加 `--debug` 参数查看原始响应，必要时联系技能维护者 |
| `detailUrl` 打不开 | cninfo 详情页需要 JS 渲染 / 链接已失效 | 用搜索引擎搜公告标题获取替代链接（交易所 PDF 直链最佳） |

> 🔁 **网络波动处理（两层保障）**：
> 1. **脚本层**：每次 API 请求失败时**自动重试最多 3 次**（指数退避 3s→6s→12s），无需人工干预。若超过 30% 的请求仍失败，打印 `[提示] 结果可能不完整` 警告。
> 2. **AI 层**：看到 `[提示]` 后，应主动重新执行一次搜索命令。如果两次结果一致，说明确实是该关键词的真实结果量；如果第二次明显增多，用第二次的结果出报告。

#### ⭐ 案例数量要求（重要）：**每次检索目标 ≥ 20 个相关案例**——案例多了才有统计意义与对照价值。凑不够 20 时，按以下顺序放宽直到 ≥ 20：① 去掉 `--type` 标题限制；② 放宽时间窗（`--start/--end`）；③ 确认 `--expand` 同义词扩展已开（默认开）；④ 换更宽泛的同义检索式。若全市场该主题**真实不足 20**（如"同一事项二次告知"类极端稀缺），**如实说明并附已找到的全部结果，绝不硬凑凑数**。`search_cninfo.py` 当结果 < 20 时会自动打印 `[提示]` 警告。

### Step 5 · 结果呈现（先总览，再分析）
- **一句话总览**：概括"检索了什么（关键词 / 代码 / 时间窗）、命中多少、共性规律"。
- **核心发现**：用 2–4 条要点提炼公告中的关键信息（报告里 `core_findings` 字段）。
- **案例索引**：表列相关公告标题 / 日期 / 类型。
- **引用片段（如需要）**：摘录公告原文关键段落作证据（报告里每案 `quotes` 字段）。
- **逐案分析**：案例概要（一段话）+ 案例分析（事实脉络 / 监管认定 / 违规或亮点 / 对本公司启示）。
- **法律分析（第二色块，推荐）**：在报告 `data` 中填 `legal_analysis` 字段，于结论速览卡片之后渲染为**独立浅底"法律分析"色块（第二色块）**，依据法规解释"为何案例结果呈现这样一种分布"（如强制/自愿的法规出处与激励逻辑）。报告开头固定三块顺序：①结论速览（第一色块）②法律分析（第二色块）③局限性（第三色块，橙色调）。详见 `references/report_template.md`。
- 对话中先贴**总览**，再说明 Word 已生成；`present_files` 给出 docx。
- ⭐ **执行后必做（用户 2026-07-15 明确要求）**：每次搜索/分析任务完成后，**必须在对话框里给一段"总体情况"回复**——说明检索了什么、命中多少、结论是什么、有无局限。不要只丢一个文件链接了事。

## 特殊场景处理
| 场景 | 处理策略 |
|---|---|
| 公告类型不明确 | 同时搜年报 / 临时公告 / 监管公告，再据结果聚焦 |
| 公司名模糊 | 先 Step 1 消歧；多匹配则列出让用户确认 |
| 搜索结果为空 | 按以下**五步排查法**逐级放宽：①确认关键词无误 → ②去掉 `--type` → ③扩大时间窗 → ④确认 `--expand` 已开 → ⑤换同义词检索式。仍无果如实告知用户，并说明可能原因（该主题确实公告少 / 关键词需调整建议）。详见「FAQ Q1」。 |
| 跨市场公司 | cninfo 覆盖沪/深/北；港股(HKEX)需另寻数据源，本技能注明局限 |
| 政策法规类 | 把检索范围扩大到监管机构的公告与法律法规 |

## 重要边界
- cninfo 全文检索 API：`http://www.cninfo.com.cn/new/fulltextSearch/full`（POST, `searchkey`/`pageNum`/`pageSize`/`sortName=pubdate`/`sortType=desc`），返回 `announcementId` 清单，**不返回 PDF 直链**（旧 static 直链 2026-05 起失效）。**关键限制：API 仅索引标题+摘要，不索引 PDF 正文**——"公司章程""董事会秘书工作制度"等上网文件的正文条款须靠搜索引擎全文或下载 PDF 提取。
- **自动重试机制（2026-07-16 新增）**：网络请求失败时，脚本会**自动重试最多 3 次**（指数退避：3秒→6秒→12秒），无需手动重新执行。若超过 30% 的请求仍失败，会打印 `[提示] 结果可能不完整` 警告。
- 公告详情页 / 列表页 / 章程 detailUrl 为 JS 渲染，WebFetch 可能只返回标题；全文优先走搜索引擎或交易所 PDF 链接。
- **"公告"≠"结尾带'公告'二字"**：公司章程、治理准则、工作细则等上网文件也是披露文件，检索"公司章程里怎么写"时必须纳入，且要进正文抠条款。
- 易董无法自动登录，相关案例须用户粘贴文本 / 链接。
- 法规判断仅供参考，重大合规决策结合最新法规与律师意见。
- 报告里的"原文链接"务必用检索到的真实 `detailUrl` / 交易所 URL，不要编造。

## 常见问题 FAQ（必读，解决 90% 的使用困惑）
> 评测反馈：重要限制和边界不要埋在流程文字里，单独列出来才不会被忽略。

### Q1：搜不到结果怎么办？
**按顺序排查：**
1. 确认关键词没写错（尤其是公司名、法规术语）
2. 去掉 `--type` 标题过滤（可能标题不含该词但摘要含）
3. 放宽时间窗口（`--start/--end` 扩大范围）
4. 确认 `--expand` 同义词扩展已开（默认就是开的）
5. 换同义词检索式（如"回购"→"股份回购/回购注销/定向回购"）
6. **以上都不行 → 如实告知用户"该主题在巨潮网上可检索到的公告确实很少"，附已找到的全部结果。绝不硬凑。**

### Q2：报错"请求失败"是什么原因？怎么处理？
| 排查顺序 | 操作 | 判断 |
|---|---|---|
| 1 | 等 10–30 秒后重试同一命令 | 如果成功了 → 就是网络临时抖动 |
| 2 | 打开浏览器访问 `cninfo.com.cn` | 如果打不开 → 是网络/DNS 问题，不是技能问题 |
| 3 | 换个关键词再试（如 `"行政处罚"` → `"处罚"`） | 如果别的关键词能搜到 → 原关键词触发了 API 的特殊限制 |
| 4 | 连续 3 次不同关键词都失败 | 切换 WebSearch 兜底：`site:cninfo.com.cn "<关键词>"` |

### Q3：为什么搜公司章程正文搜不到具体条款？
这是 **API 的已知限制**（不是 bug）：巨潮全文检索只索引标题+摘要，不索引 PDF 正文。
- **能做的**：用脚本搜到章程文件清单（全市场 14.7 万份），拿到文件链接
- **然后**：下载 PDF 提取正文（PyMuPDF）或用搜索引擎抠条款
- **已在文档中说明的位置**：本文件「章程 / 上网文件检索」章节 + 「能力边界一览表」

### Q4：港股 / 新三板 / 美股能搜吗？
- **港股 (HKEX)**：❌ cninfo 不覆盖，需另寻数据源（如阿斯达克 aaStocks、港交所披露易）
- **新三板基础层**：⚠️ cninfo 覆盖有限，部分公告可能在股转系统
- **美股**：❌ 不支持
- 报告中会注明此局限（第三色块"局限性"）

### Q5：结果少于 20 个怎么办？
- 先按 Q1 流程放宽检索条件
- 如果全市场该主题**真实就不足 20**（比如"同一事项二次告知"这类极稀缺主题）→ **如实说明，附全部已有结果**
- 脚本会在结果 < 20 时打印 `[提示]` 警告，不会静默返回

### Q6：网络波动导致结果不全怎么办？
- **脚本已内置自动重试**：每次 API 请求失败会自动重试最多3次（3s→6s→12s退避），不需要手动干预
- **何时需人工重跑**：当控制台打印 `[提示] 结果可能不完整`（>30%请求失败）时，说明网络确实不稳定，应重新执行同一命令
- **判断标准**：如果两次结果一致 → 确认为真实结果量；第二次明显增多 → 用第二次的结果出报告

### Q7：生成的 Word 报告格式不对 / 打不开？
- 确认已安装 python-docx：运行 `pip install python-docx`
- 确认 Python 版本 ≥ 3.8
- 如果报告生成但内容为空：检查输入 JSON 文件是否完整（`--json` 参数指定的文件）

### Q8：想用自己的数据源（如 Wind / 易董）代替 cninfo？
- **Wind**：有 API 但需机构账号；本技能的查询拆解方法论（五步法、同义词扩展表）可直接复用到 Wind 检索词构建上
- **易董**：需用户手动登录后导出/粘贴案例文本；本技能的"易董补录模式"支持对粘贴内容做同样格式的报告加工
