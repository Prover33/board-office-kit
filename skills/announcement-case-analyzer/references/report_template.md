# 案例分析报告模板与写法

## 输出顺序（用户要求：结论置顶、一眼看懂；再总览、再分析；参考 Wind 五步法 Step 5）
1. 标题：**上市公司公告案例分析报告**（居中）
2. 副标题：本次主题
3. 元信息行：检索范围 / 案例类型 / 数据来源 / 生成日期（可附 `query` 实际检索式、`time_window` 解析出的时间窗，便于复现）
4. **★结论速览卡片（第一色块，置顶，必填）**：浅蓝底带框卡片，图标化，让领导**一眼看懂**。含：
   - ◆ 主题（读 meta.topic）
   - ● 案例数（自动统计 len(cases)，≥20 标绿"满足下限"、<20 标橙"低于下限"）
   - ★ 核心结论（读 `verdict` 字段，红色加粗，**务必一句话说清结论**；缺省时自动取 conclusion/overview 首句兜底）
   - → 关键规律（自动取 core_findings 前 3 条）
   - **不含局限性**（局限性移到第三色块）
5. **法律分析色块（第二色块）**：紧跟结论速览之后，独立浅底卡片（读 `legal_analysis` 字段）。**依据规则解释"为何案例结果呈现这样一种分布"**，如强制/自愿的法规出处与激励逻辑；无则省。
6. **局限性色块（第三色块）**：紧跟法律分析之后，独立浅底卡片（橙色调，读 `limitations` 列表；缺省时取 note 兜底）。**如有局限务必写，没有可省**。
7. **总览**：一段话概括检索范围、命中数量、共性规律（读 `overview`）
8. **核心发现**：2–4 条要点（读 `core_findings`，可选）
9. **一、案例索引**：表格（#、公司、代码、公告标题、日期）
10. **逐案例展开**：基本要素 → 【案例概要】 → 【案例分析】 → 【监管视角 / 可借鉴点】（可选） → 【引用片段】（可选） → 原文链接
11. **二、总体结论**（结论详述，与速览的一句话呼应）
12. 附注（数据来源说明）

## 各段写法要点
- **总览（一段话，置顶）**：本次检索了什么（关键词 / 代码 / 范围）、命中多少、这些案例的**共性规律或核心发现**是什么。给领导一眼看懂。
- **核心发现（2–4 条要点，可选）**：把"最值得注意的结论"单列，便于汇报直接引用。
- **案例概要（一段话）**：谁、何时、因何事项、做了什么、结果如何。信息密度高。
- **摘要客观性规范（必守）**：摘要必须**来自公告事实、客观表述**——包含关键议案、金额、比例、交易对手、出席股东人数、持股比例等明确事实；**不加入推断或主观评价**，不写"可能""涉嫌"（除非公告原文如此表述）。同一家公司多条公告分别用多个「公告标题」小节。
- **案例分析**：①事实脉络 ②监管认定 / 关注点 ③违规或亮点 ④对本公司（海南华铁）启示。
- **监管视角 / 可借鉴点**：把"对自己有用"的结论单点，便于汇报直接引用。
- **引用片段（可选）**：摘录公告原文关键段落作证据，增强可信度；务必来自真实抓取，不编造。
- **原文链接**：必须用检索到的真实 URL（cninfo `detailUrl` 或交易所），禁止编造。
