# 检索方法（巨潮网 cninfo 全文检索为主）

## 一、首选：cninfo 全文检索 API（脚本化、可复现）
技能自带 `scripts/search_cninfo.py`，调用巨潮网全文检索接口：
`POST http://www.cninfo.com.cn/new/fulltextSearch/full`
参数：`searchkey`(关键词/代码) `pageNum` `pageSize` `sortName=pubdate` `sortType=desc`
返回 JSON 含每篇公告的 `announcementId` / `secCode` / `secName` / `announcementTitle` / `announcementTime` / `detailUrl`。

用法：
```
PYTHON="C:/Users/admin/.workbuddy/binaries/python/envs/default/Scripts/python.exe"
SKILL="C:/Users/admin/.workbuddy/skills/announcement-case-analyzer"
"$PYTHON" "$SKILL/scripts/search_cninfo.py" "<关键词>" --pagesize 10 --json /tmp/cninfo.json
```
- ⚠️ **重要限制（实测）**：cninfo 全文检索**只索引公告标题 + 摘要，不索引 PDF 正文全文**。实测佐证：搜 `陈述申辩`（几乎每份告知书正文都有）全市场仅 4 条（均为标题含“陈述申辩意见”的申辩公告），而 `行政处罚事先告知书`（正文普遍出现）有 1041 条。故**只出现在正文的区分词（如“前次”“再次”“同一事项”“分别收到”）在 cninfo 恒为 0 命中**，这类需求必须走下方 WebSearch 全文索引。
- 关键词命中范围：标题/摘要含该词即命中（如 `行政处罚` `违规担保` `抽屉协议` `业绩承诺` `控制权转让`）。
- **直接传 6 位代码**可搜某公司全部公告（如 `603300` 搜海南华铁）。
- **支持 cninfo 布尔/短语**：空格 = AND、`OR`、`NOT`、引号 = `"精确短语"`（如 `"控股权转让协议"`）。

## 三、兜底：WebSearch（当 API 不可达）
- `site:cninfo.com.cn "<关键词>"` 可叠加代码 / 年份 / 类型词。
- 一手源：`site:sse.com.cn` / `site:szse.cn` / `site:bse.cn`。
- 取全文：搜精确标题找 `dataclouds.cninfo.com.cn/xc2pdf2h5/...` HTML5 链接再 WebFetch。

## 四、特殊难搜类型：同一事项“第二次收到行政处罚事先告知书”
该类型极难搜，原因是：① 公告标题与首次告知书往往**完全一致**（都叫“关于收到行政处罚事先告知书的公告”），“第二次”信号只藏在正文；② 巨潮 API 不索引正文（见上）；③ A 股中该事件本身稀少——《证券期货违法行为行政处罚办法》第三十二条：监管对已送达告知书认定的事实/依据/拟处罚决定作出调整且**非有利变更**的，应重新送达告知书（有利变更则直接出决定书、无二次告知）。
- **法定依据**：《证券期货违法行为行政处罚办法》（证监会令第186号）第三十二条。
- **唯一可行路径**：WebSearch 走搜索引擎的 PDF 全文索引（巨潮 API 无效），检索式（配合 `site:cninfo.com.cn`）：
  1. `前次《行政处罚事先告知书》 本次`
  2. `原《行政处罚事先告知书》 变更 重新送达`
  3. `分别收到 行政处罚事先告知书`
  4. `重新向你送达 行政处罚事先告知书`
  5. `（二次） 行政处罚事先告知书 上市公司`
- **已验证的最贴近实例**：先锋新材（300163）2026-05-15 同日收到甬证监告字〔2026〕4号（致实控人）与 5号（致公司及相关当事人），均指向 2017 年“未披露为关联方提供担保”同一事项——即“同一件事、两份告知书”（巨潮原文见 announcementId=1225311527）。严格“先一份、隔段时日再一份”的先后型 A 股样本，自动检索暂未捞出干净案例，需结合具体公司名定向深挖。

## 五、章程 / 上网文件检索（进正文抠条款，不是只搜"公告"）
用户问"公司章程里怎么写 XX"时，目标文件是**上网文件**：公司章程、董事会秘书工作制度、总经理工作细则、提名委员会议事规则等，在巨潮网/交易所归档为独立披露文件（标题多为"XX公司章程（YYYY修订）"）。**结尾不是"公告"二字也是披露文件，务必纳入检索。**

- ⚠️ cninfo API 只索引标题+摘要、不索引 PDF 正文（见一），故 `search_cninfo.py "公司章程"` 能按标题捞出章程文件清单（全市场 14.7 万份），但**拿不到正文条款**——别误判"搜不到"即不存在。
- **抠条款两条路（优先 a，最快）**：
  - (a) **搜索引擎全文索引**（已爬 PDF 全文）：直接用条款原文短语 + `公司章程` 检索，返回含条款摘录与原文链接的结果。例：
    - `"董事会秘书由董事长提名" 公司章程`
    - `"董事会秘书由总经理提名" 公司章程`
    - `"董事会秘书由董事会提名" 公司章程`
    - `公司章程 "董事会秘书" "董事长提名" "董事会聘任"`
  - (b) **下载 PDF + PyMuPDF 提取**：先用 `search_cninfo.py "公司章程"` 或搜索引擎捞出章程 `detailUrl`/PDF 直链，再用 `pymupdf` 提取正文（本技能 `_hnht` 样例已验证：下载 54 页章程 PDF → `fitz.open` → `get_text` → 定位"第六章 高级管理人员"）。cninfo `detailUrl` 是 JS 渲染页，WebFetch 仅返回标题，须取真实 PDF 直链（如 `notice.10jqka.com.cn/api/pdf/...`、`static.cninfo.com.cn/finalpage/...PDF`、交易所 `sse.com.cn/disclosure/...pdf`）。
- **区分"公司章程"与"工作制度/细则"**：制度系"依据《公司章程》制定"，其董秘提名等条款与章程一致，可作章程约定旁证；报告须注明每条是"公司章程"原文还是"工作制度（引章程）"。
- **上位规则（抠条款时一并核对）**：证监会《上市公司董事会秘书监管规则》(公告〔2026〕8号，2026-05-24 施行，**第二十三条**：董事会秘书由董事会聘任，**董事会提名委员会对董秘人选及其任职资格进行遴选、审核**)；《上市公司治理准则(2025年10月修订)》第四十七条（提名委员会审核高管人选）；《上市公司章程指引》。
- **已验证实例（董秘提名权）**：海南华铁(603300)章程(2025.12)第一百四十九条**仅写董秘职责、未写提名权**（留白）；同业主流 17 家写"董事长提名、董事会聘任"；科森科技(603626)为"董事长提名+提名委员会审查"；688439 将"总经理由董事会提名"。详见 `公告案例分析_章程董秘提名权_20260715.docx`。

## 六、易董补录模式
用户从易董导出 / 复制的案例，直接粘贴标题、原文段落、链接即可，无需检索。
