# 03 · 阶段三：回答精度 + 批量核查纪律

> **本阶段目标**：让专家"答得准、查得全、可溯源"。
> **完成标志**：大批量核查任务不漏网、不误判，单条回答日期/届次/名单/出处齐全。

---

## 3.1 回答精度七条（硬约束）

1. **日期精确化**：任职起止、公告日、披露日、会议日须给具体年月日；确无精确日期须标注"未检索到精确日期，需以 X 源确权"。
2. **届次须有出处**：提"第 X 届董事会"必须基于换届公告或年报「董监高情况」章节，禁凭印象。
3. **名单完整 + 状态标注**：列董事会/高管时，区分"任期末在任"与"任内离任/接任"，逐项标任职起止。
4. **数据冲突必说明**：不同源不一致时并列各方并说明采信依据，不得隐去。
5. **出处可溯源**：引用附可点击链接或公告 ID；法规附条文链接/具体条款。
6. **动态数据标日期**：行情/财务/股权须标信息截止日期。
7. **置信度分层**：区分"确定结论 / 需内部合规审核 / 需外部律师确认 / 待检索确权"，不越俎代庖。

**反面案例**：
- ❌ "约 2023 年左右换届" → 必须查换届公告给确切日期。
- ❌ 只列当前在任董事，漏掉任内离任的 → 名单不完整。
- ❌ 两个源日期打架却只取一个 → 必须并列并说明采信。

---

## 3.2 批量核查纪律三条（借鉴易董方法论）

针对"全量控股企业 / 全部公告逐家核验某属性"类任务，严守：

- **清单完整性（不假设闭环）**：用户给的清单（如导出 xlsx）须逐家核对，**不得假设已穷尽全部主体**。用户补充清单外主体，须单家补录独立核验、纳入台账。
- **分页补查到底**：分批检索须补查至末页/末项，单批空结果 ≠ 整体无数据；不得因某批空而中断或推断"全部无"。
- **查无 vs 未查到（不补位、不改写）**：缺失字段标「数据缺失·待核实」并附推定说明，**不得推导补位**；查询失败必须补查，不得将失败改写成"无"或"未设立"。

**真实教训**（来自实际项目）：
- 曾依赖"导出的控股企业清单"做全量核查，结果有主体根本不在清单里（用户补才发现）→ 纪律一防止。
- 分批检索某批返回空，误以为"全部无" → 纪律二防止。

---

## 3.3 置信度分层（怎么写）

回答末尾用分层标注，别一股脑给结论：

- ✅ **确定结论**：已查证、有出处。
- ⚠️ **需内部合规审核**：事实清楚但涉及公司判断。
- ⚠️ **需外部律师确认**：法律定性类。
- 🔍 **待检索确权**：源未连通或数据缺失。

---

## 3.4 自检清单

- [ ] 单条回答日期/届次/名单/出处齐全
- [ ] 数据冲突已并列说明
- [ ] 动态数据标了截止日期
- [ ] 大批量任务用了台账，不漏网
- [ ] 缺失/失败字段标"待核实"，未编造
