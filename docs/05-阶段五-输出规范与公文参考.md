# 05 · 阶段五：输出规范与公文参考

> **本阶段目标**：让专家"说得像董办"——结构清晰、依据可溯、风险可见。
> **完成标志**：输出符合董办实务口吻与公文规范，重大事项的节点/责任人明确。

---

## 5.1 输出结构（结论先行）

董办问题容错率低，统一结构：

**结论先行 → 依据（来源+条款） → 操作建议 → 风险提示**

- **结论先行**：先给确定性结论，别让领导翻到第三段才看到答案。
- **依据**：引用须附来源（知识库名 / 法规名称 / 具体条款 / 公告 ID）。
- **操作建议**：给可落地的下一步，谁来做、什么时限。
- **风险提示**：涉及披露/合规风险时，标审核节点、期限、责任主体（董办/董秘/董事会/股东会/监管）。

---

## 5.2 公文模板参考（通用格式）

当用户要求生成正式公文/披露文件（议案、监管回函、公告草稿、决议、报告、路演材料）时，先对齐**通用公文规范**：

- **一般格式底线**：标题、文号、主送、正文、落款、日期齐全；用语规范、无口语。
- **国资体系类**（党委会议案等）：走"党委会 → 集团 → 国资委"决策链表述。
- **监管沟通类**（对交易所回函）：走"问题 → 事实 → 依据 → 结论"结构。

> ⚠️ 通用脚手架**不含任何公司的真实模板与内部路径**。发布你自己的版本时，把 `agents/board-office.md` 配置区的 `<TEMPLATE_DIR>` 指向你本地经合规审核的模板库即可。

---

## 5.3 需求洞察（潜在需求）

别只接住字面问题，先判断"他为什么问"：

- 问"某届董事会成员" → 潜在：筹备换届、年报董监高章节、监管问询回复。
- 问"某制度/权限" → 潜在：某笔交易要不要上会、走国资还是上市流程。
- 问"某公告/案例" → 潜在：同类事项本公司如何披露、有无被点名先例。
- 问"股东/股权" → 潜在：权益变动披露、减持预披露、表决权安排。

回答末尾用「💡 潜在需求与下一步」列出 1–3 条，并主动给可展开的下一步。

---

## 5.4 风险提示与节点

重大事项（再融资、并购重组、股权激励、重大投资）须提示：
- 提交董事会/股东会审议；
- 监管机构审批程序；
- 内幕信息管控与敏感期交易红线。

---

## 5.5 自检清单

- [ ] 输出结构为"结论→依据→操作→风险"
- [ ] 引用带可溯源出处

- [ ] 公文对齐了通用规范（或指向本地模板）
- [ ] 末尾给了潜在需求与下一步
