# 01 · 阶段一：搭框架（配置区 + 角色定位 + 核心能力）

> **本阶段目标**：把通用骨架变成"你的"专家——填好配置区、定好立场与目标、列清三大能力、说清治理架构与双体系边界。
> **完成标志**：专家被问"你是谁 / 你能做什么"时，能准确报出目标公司身份与能力边界，不串台、不编造。

---

## 1.1 为什么先搭框架

董办专家最怕两件事：**串台**（用工商源答法规问题）和**编造**（没数据硬编）。框架阶段就是给专家"装上身份 + 戴上边界"，让它知道自己是谁、能碰什么、该去哪查。这一步不做，后面接再多 MCP 也是空中楼阁。

---

## 1.2 填【配置区】（最关键的一步）

打开 `agents/board-office.md` 顶部的【配置区】，逐项把 `<占位符>` 换成目标公司真实信息。**填完前专家不可用。**

| 配置项 | 怎么填 | 填错会怎样 |
|---|---|---|
| 公司全称 `<TARGET_CO>` | 营业执照全称，别用简称 | 对外回复张冠李戴 |
| 证券代码 `<TICKER>` | 交易所+代码，如 `SH600000` | 行情/公告路由错乱 |
| 上市板块 `<BOARD>` | 主板/科创板/创业板/北交所 | 适用的上市规则不同 |
| 是否国资控股 `<STATE_OWNED>` | 是/否 | 决定要不要"双体系"约束 |
| 国资审批链（如适用） | 党委会→集团→上级党组织→国资监管机构（按实际） | 漏写则交叉事项不双线提示 |
| 治理架构 `<GOVERNANCE>` | 单层制（董事会+审计委员会，不设监事会）/ 双层制（董事会+监事会） | 监事会问题会答错 |
| 知识库（可选）`<KB_ID>` | 自建 IMA/笔记库 ID | 制度问答无源可查 |
| 公文模板目录（可选）`<TEMPLATE_DIR>` | 本地议案/回函模板路径 | 出公文时无规范可对齐 |

**填写范式**（以某制造业沪市主板国资控股为例）：

```
公司全称 = 某某智造股份有限公司
证券代码 = SH600XXX
上市板块 = 主板
是否国资控股 = 是
国资审批链 = 党委会 → 集团党委 → 省国资委
治理架构 = 单层制（董事会下设审计委员会，不设监事会）
知识库 = <你的 IMA 库 ID>
公文模板目录 = D:/.../董办/模板/
```

> ⚠️ 占位符必须**全部**替换完，且不得残留任何示例公司的字段（业务板块、行业地位等下文也要同步改）。

---

## 1.3 定立场与目标

配置区之下，确认三段定位语（骨架已给，按公司微调）：

- **立场**：站在上市公司董办 / 董事会秘书一侧，服务合规运作、信息披露与公司治理。
- **目标**：把监管规则、公司内部制度与目标公司的具体业务结合，输出"说得清依据、做得到落地"的解答。
- **检索优先**：能用知识库与公开权威源查到的，不凭空编造；查不到的，明确说明并给获取路径。

---

## 1.4 列三大能力类（按需裁剪）

骨架给了三类能力，按目标公司实际**保留/增删**：

1. **证券事务专长**：法规体系、信息披露、IR、资本运作、监管合规红线。
2. **董秘职责全覆盖**：董事会/股东会、治理、信披管理、投资者沟通、股权管理、资本运作支持。
3. **目标企业知识**：公司背景、业务板块、战略定位、行业地位、治理双体系（如适用）。

> 实操建议：业务板块、战略定位、行业地位这三行**必须换成自家真实披露口径**，别留示例痕迹。

---

## 1.5 说清治理架构与双体系

- **治理架构**：单层制（董事会 + 审计委员会，不设监事会）还是双层制（董事会 + 监事会）？这决定你答"监事会"类问题时是"不设"还是"有专门委员会"。
- **双体系边界**（国资控股才需要）：国资监管体系（党委会→集团→国资委）与上市公司治理规则（股东大会→董事会→委员会）**不能混淆**；交叉事项须双线提示，标注"⚠️ 国资体系审批流程请查询权限指引手册"。

---

## 1.6 自检清单

- [ ] 配置区无残留 `<占位符>`
- [ ] 业务板块/战略/行业地位已换成自家口径
- [ ] 治理架构描述与实际一致（单层/双层）
- [ ] 国资公司已在"双体系"处标注边界
- [ ] 用"你是谁、你能做什么"两个问题测试，专家答得准

---

## 1.7 常见坑

- ❌ 直接照搬示例公司的业务板块 → 必须替换为自家，否则对外回复全是别人家的事。
- ❌ 国资公司漏写双体系 → 交叉事项（担保/投资上会）会混淆两条审批链。
- ❌ 配置区填完不验证 → 先拿"你是谁"测一下，确认身份绑定成功再往下走。
