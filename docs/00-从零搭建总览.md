# 00 · 董办专家方法论总览（skill 内部知识）

> ⚠️ 本文件是 **board-office-kit** skill 的**内部方法论知识**，供 AI 在运行时检索与对齐，**使用者（董办同事）无需阅读**。
> **技能定位**：board-office-kit 是一个**通用、完整的董办专家 skill**。使用者只需在 `agents/board-office.md` 的【配置区】填入自己公司的信息，它即成为"属于自己公司的董办专家"，可直接投入日常实务，**无需从零搭建**。

本总览把 skill 的方法论拆成五个阶段（框架 → 路由+MCP → 精度+纪律 → 进化 → 输出），便于 AI 在对应环节检索对齐；各阶段详写在 `docs/01`~`docs/05`。

---

## 一、为什么需要分阶段

董办专家的核心价值不在"会聊天"，而在三件事：
1. **场景路由准**——问工商的去工商源、问法规的去法规源，不混用。
2. **数据可溯源**——每条结论能附出处、日期、置信度。
3. **持续能进化**——新数据源/新坑位能回流成规则，下次直接命中。

这三件事没法一步到位，所以拆成五个阶段，每阶段都有明确交付物，搭完一段就能用一段。

> **设计原则：开箱即用 + 越搭越准。** 专家零配置即可干通用董办活；填【配置区】绑定身份、建 IMA 知识库沉淀内部制度、接 MCP 接入权威源——每多搭一层，越懂你家公司、越精准。不要等"全配完"，先让它干活。

## 四、推荐的搭建顺序（不要跳步）

1. **（可选）先开箱即用**——即使不填配置区，专家也能以"通用 A 股董办"身份直接干活；想从"通用"变"你的"，再填【配置区】。
2. **建 IMA 知识库**（越填越精准）——把公司治理制度、权限审批表传进 IMA，制度/权限问答直接命中内部规定；专家还能回写补库。这是"懂你家公司"的关键一步。
3. **再接 MCP**（阶段二）——没有数据源，专家只是空壳。每接一个就在 `mcp-wiring-checklist` 打勾。
4. **然后定精度与核查纪律**（阶段三）——尤其要做全量核查时，纪律决定成败。
5. **最后做进化与输出**（阶段四、五）——让专家越用越聪明、越说越像董办。

## 五、一个实战教训（为什么需要"批量核查纪律"）

在真实项目中，曾出现两类典型失误，本脚手架已固化为纪律防止重犯：
- **清单漏网**：依赖"导出的控股企业清单"做全量核查，结果有主体根本不在清单里（用户补才发现）。→ 纪律一：不假设清单闭环，用户补充的清单外主体须单家补录核验。
- **空结果误判**：分批检索时某批返回空，误以为"全部无"。→ 纪律二：分页补查到底，单批空 ≠ 整体无；纪律三：查询失败必须补查，不得改写成"无"。

这三条是易董等成熟专家方法论的共性沉淀，已写入 `agents/board-office.md` 的「大批量核查与数据完整性纪律」段。
