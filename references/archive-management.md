# 档案文件管理方法论（内置能力）

> 本文件是专家内部的方法论知识（供 AI 检索），使用者无需阅读。
> 适用场景：用户想"理顺电脑里存放的董办相关文件"——扫描建档、归类、写目录、后续快速找文档 / 帮存放新文件 / 帮编辑。

---

## 一、能力定位

董办日常积累大量文件：公告底稿、会议议案、制度 PDF、对交易所回函、合同、调研纪要、财务底稿等。文件散落、命名随意、重复堆积，是普遍痛点。本能力的目标是：

1. **先看清**——只读扫描目标目录，生成一份可溯源的文件清单（不看内容也能量体裁衣）。
2. **再理顺**——基于文件名、类型、修改时间，建议归入哪个主题桶；**主动询问用户**是否执行归类。
3. **留目录**——生成可读的「目录索引」文件，后续一眼定位某类文档在哪。
4. **能复用**——把归档结构与归类偏好记下来，之后可：帮用户把**新文档存到正确位置**、**快速找到某类文档**、**协助编辑**（路由到本地 Office 编辑能力）。

---

## 二、安全约束（铁律，违反即事故）

文件操作属于高风险动作，本能力**以"只读优先、动作必确认"为唯一准则**：

1. **扫描 = 只读**：扫描 / 列目录 / 生成清单，全程**不移动、不重命名、不删除任何用户文件**。
2. **个人目录高危**：目标为桌面 / 下载 / 文档 / Home 或个人根目录时，视为**高危目录**——
   - 只出报告（路径 / 大小 / 日期 / 类型 / 推测主题）；
   - **绝不自动做任何改动**；任何移动/重命名/删除前，必须**逐批**列出受影响路径、说明风险，获用户明确确认后才执行（每批 ≤ 10 个文件）；
   - 删除必须走回收站 / 先备份，禁止 `rm -rf`、禁止静默销毁。
3. **项目 / 业务档案目录**（如董办工作区、业务档案库）：可执行归类，但**任何移动/重命名前仍须用户确认**；先生成「归类方案」让用户拍板，再动手。
4. **不改写正文**：归类只动"位置/命名"，不动文件内容；需要编辑内容时走专门的文档编辑流程（见第五节）。
5. **敏感信息**：扫描到的文件可能含未公开内幕信息 / 内部制度，索引中**只记录元数据（路径/大小/日期/主题标签），不记录文件内容摘要**，避免泄露。

## 三、四步流程

### 步骤 1 · 只读扫描建档（Scan，只读）

- 先**向用户确认目标目录**（必须明确指定，不接受"整个电脑"这类模糊范围）。
- 递归列举文件，采集每个文件的：`相对路径`、`大小`、`修改日期`、`扩展名/类型`。
- 轻量推断主题：依据文件名关键词 + 扩展名，打**初步主题标签**（公告 / 会议议案 / 制度 / 回函 / 合同 / 底稿 / 财务 / 其他）。
- 检测**重复 / 近似文件名**（同名不同路径、带"(1)""副本"后缀），单独列出供用户决策。
- 产出：`档案扫描报告.md`（仅元数据清单，不碰原文件）。

### 步骤 2 · 归类建议（Ask，主动询问）

- 基于步骤 1 的主题标签，提出**归类方案**：建议的目标目录结构（如 `01-公告底稿/`、`02-会议议案/`、`03-制度/` …）与每个文件的归属。
- **主动询问用户**：是否执行归类？采用哪套目录结构？哪些文件保持原位？
- 用户决策前**不落任何动作**；用户选择"只建目录不移动"时，仅生成目录索引文件。
- 记录用户的归类偏好（如"回函统一放 04-监管回函"），作为后续复用依据。

### 步骤 3 · 生成目录索引（Index）

- 生成两份索引，放在目标目录内（或用户指定位置）：
  - `档案目录.md`：人类可读的分层目录 + 每类文件计数 + 最后更新日期。
  - `档案索引.json`：机器可读，每条含 `路径 / 大小 / 修改日期 / 主题标签 / 归类桶`，供后续"快速找文档"检索。
- 索引本身也是文件，写入前仍须用户知晓存放位置（项目目录直接写；个人目录须确认）。

### 步骤 4 · 记忆与复用（Remember & Reuse）

把以下内容沉淀下来，使专家"越用越懂你的档案"：

- **归档结构 & 归类规则**：记到 IMA 知识库（如「<TARGET_CO> 档案归类规范」）或用户确认的本地产地；之后新文件进来，专家能直接建议落点。
- **常用目录约定**：如"公告底稿在 `01-公告底稿/`""回函在 `04-监管回函/`"。
- **后续三种复用**：
  1. **快速找文档**：用户说"上次那个对交易所的回函"，专家查索引定位路径并打开。
  2. **帮存放新文档**：用户丢来一份新文件，专家按归档规则建议目标路径，确认后存放（仍走确认流程）。
  3. **协助编辑**：需要改内容时，路由到本地 Office 编辑能力（tencent-local-office-edit 等），所见即所得。

## 四、目录索引格式（示例）

`档案目录.md` 示例：

```markdown
# <目录名> 档案目录

> 最后更新：2026-07-20 · 共 138 个文件 · 由董办专家自动编目

## 01-公告底稿（42）
- 2026-临时公告-关于增资的议案.docx
- 2026-年度报告.docx
（…）

## 02-会议议案（18）
（…）

## 03-制度（11）
（…）

## 04-监管回函（7）
（…）

## 其他 / 待归类（12）
- 未命名扫描件.pdf  ← 建议人工确认主题
```

`档案索引.json` 示例（节选）：

```json
[
  {
    "path": "01-公告底稿/2026-临时公告-关于增资的议案.docx",
    "size": 88421,
    "mtime": "2026-07-18",
    "type": "docx",
    "tag": "公告",
    "bucket": "01-公告底稿"
  }
]
```

## 五、与现有能力衔接

- **扫描 / 列目录**：使用环境提供的目录列举能力（不读正文内容）。
- **编辑文件内容**：路由到本地 Office 编辑能力（如 tencent-local-office-edit），**不改写归档结构**，仅编辑文档正文。
- **制度类归档依据**：若某文件应按公司内部制度归类，命中 IMA 知识库中的「档案归类规范」。
- **安全兜底**：任何写动作（移动/重命名/写索引/删除）前，执行第二节安全约束；个人目录一律走高危流程。

## 六、一次完整交互长什么样

> 用户：帮我把董办工作区里的文件理一理。
>
> 专家：
> 1. （确认目录后）只读扫描，产出《档案扫描报告》：共 138 个文件，初步分为公告/议案/制度/回函/其他 5 类，发现 3 处重复文件。
> 2. 给出归类方案：建议建 `01-公告底稿/` `02-会议议案/` `03-制度/` `04-监管回函/` 四桶，并列出每类文件清单。
> 3. 问：要我直接帮你移动归类，还是先只生成目录索引、文件保持原位？（个人目录会特别强调"先确认再动"）
> 4. 用户选"生成目录 + 移动"→ 专家先写 `档案目录.md` + `档案索引.json`，再逐批（≤10/批）移动并回报，绝不删原文件、不碰内容。
> 5. 把"归档结构 + 归类规则"记到 IMA 库，后续新文件进来可直接建议落点。
