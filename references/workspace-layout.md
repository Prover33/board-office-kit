# 工作区规划与文件归位方法论

本文件定义董办专家"工作区"的标准分区，以及生成文件应按主题归位到哪个分区。目标是：用户第一次使用时就能规划好一个专属工作区，专家后续产出的所有文件自动落到对应分区，**顶层只留成品，中间物与脏数据隔离在缓存区**，避免一大堆文件堆在根目录、难找难管。

---

## 一、何时规划工作区

- **首次交互**（尤其【配置区】尚未填写或 `<WORKSPACE>` 为空时），专家**主动提醒**用户规划一个工作区根目录，并展示下方标准分区方案，请用户确认根路径（或自定义）。
- 建议默认根：当前对话所在项目目录下建 `董办工作区/`；若用户指明其他路径，以用户指定的为准。
- ⚠️ 目标为桌面/下载/文档/Home 等**个人目录**时视为高危：先说明风险，确认后再建；不擅自在个人目录下铺文件夹。

---

## 二、标准分区（带序号，保持顺序）

```
<WORKSPACE>/
├── 00-方法知识包/      # 方法论、速查卡、搭建笔记、专家导出的知识包（可复用）
├── 01-公文草稿/        # 公告、回函、议案、公文骨架（待审稿 / 终稿）
├── 02-核查台账/        # 子公司核查、董事任职追溯、全量台账（CSV / MD）
├── 03-案例与法规/      # 检索到的公告案例（带原文链接）、法规摘录、处罚案例
├── 04-知识库/          # IMA 库导出的制度、权限审批表、搭建产物
├── 05-会议记录/        # 三会（股东大会/董事会/专门委员会）会议材料、决议记录
├── 06-信披材料/        # 定期报告草稿、临时公告底稿、披露事项清单与时间表
├── 07-治理文件/        # 公司章程及修正案、议事规则、内控制度汇编
├── 08-对外沟通/        # 监管函件往来、问询回复底稿、投资者关系记录
├── 09-档案目录/        # 档案扫描报告、档案索引（来自档案文件管理）
└── ZZ-工作缓存(可删)/  # 中间文件、临时脚本、可重跑辅助（脏数据，可随时清空）
```

编号约定：
- `00-` 放方法论/知识包类；
- `01-`~`09-` 按主题分桶的成品区，**连续编号无跳号**；
- `ZZ-工作缓存(可删)/` 统一收口临时物，名字含"可删"提醒无负担清理。

---

## 三、生成文件 → 分区的路由规则

专家每次落盘文件前，先判断产物类型，写入对应分区：

| 产物类型 | 落盘分区 | 说明 |
|---|---|---|
| 起草的公告 / 回函 / 议案 / 公文骨架 | `01-公文草稿/` | 含待审稿与终稿；终稿建议文件名加 `(终稿)` 或日期后缀 |
| 子公司全量核查台账、董事任职追溯、控制企业清单 | `02-核查台账/` | CSV + 同名 MD 说明一并放；台账注明核查日期与源 |
| 公告案例检索报告、监管/处罚案例、法规摘录 | `03-案例与法规/` | 案例报告带原文链接；法规摘录附条文号 |
| IMA 库导出的制度、权限审批表、搭建产物 | `04-知识库/` | 与线上库同源，本地留底便于离线查阅 |
| 三会会议材料、决议记录、委托书、签到表 | `05-会议记录/` | 按会议届次或日期分子文件夹；决议附表决结果 |
| 定期报告草稿、临时公告底稿、披露时间表 | `06-信披材料/` | 按报告期（2025年报/Q1等）或公告类型分桶 |
| 章程及修正案、议事规则、内控制度汇编 | `07-治理文件/` | 每份制度单独一个 MD/PDF，附生效日期与版本号 |
| 监管函件往来、问询回复底稿、投资者关系记录 | `08-对外沟通/` | 函件按收发对象分；回复留底稿+最终发送版 |
| 档案扫描报告、档案索引 | `09-档案目录/` | 来自"档案文件管理"能力 |
| 方法论 / 速查卡 / 专家自身知识包导出 | `00-方法知识包/` | 可复用的内部知识 |
| 临时脚本、中间 CSV、可重跑辅助产物、脏数据 | `ZZ-工作缓存(可删)/` | 不进成品区；发现可被成品替代时标注并询问是否清理 |

> **原则**：成品只在主题区（`00-`~`09-`）；任何"中间物 / 可重跑 / 试探性"产物一律进 `ZZ-工作缓存(可删)/`。顶层（`<WORKSPACE>` 根）保持干净，不做日常落盘。

## 四、归位执行纪律

1. **先确认根，再建分区**：用户未确认 `<WORKSPACE>` 前，不擅自建文件夹；确认后一次性建好标准分区（仅在该根下）。
2. **生成前确保分区存在**：落盘某分区文件时，若该分区不存在则先建（mkdir），再写文件。
3. **同名 / 旧版处理**：发现分区内已有同名文件，不静默覆盖——标注"新版本 / 旧版本"，询问用户覆盖还是另存（加日期后缀），不擅自销毁交付物。
4. **缓存与成品隔离**：临时脚本、中间产物生成时直接进 `ZZ-工作缓存(可删)/`，不污染成品区。
5. **保持可溯源**：每个落盘文件建议自带简短头部（生成日期、依据源、对应任务），便于后续检索与档案能力接管。
6. **安全边界**：仅在该工作区内读写；不移动/重命名/删除用户既有文件（见 `archive-management.md` 只读扫描铁律）；写入个人高危目录须逐批确认。

---

## 五、与"档案文件管理"的衔接

- 工作区建好后，可随时用"档案文件管理"能力对 `<WORKSPACE>` 做只读扫描，生成「档案目录.md + 档案索引.json」放进 `09-档案目录/`，形成可检索的本地档案视图。
- 档案能力发现的散落文件，可建议用户按本分区方案归位（仍须逐批确认）。
