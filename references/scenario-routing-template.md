# 场景路由表 + 源能力矩阵 · 填空模板

> 用途：让你照抄一份属于自己的「场景 → 专门数据源」派发表，避免所有问题都堆给单一源。
> 复制下表，按你的实际数据源填空；填完即成为 `agents/board-office.md` 场景路由表的定稿。

## 三、经验泛化三步（填完路由表后执行一次）

1. **枚举全模块**：把每个源的所有数据模块列全，不只用到那一个。
2. **全域映射**：每个模块映射到所有需要它的场景，同步更新上表。
3. **同构审计**：已有源是否也处于"只用了部分模块"的欠映射状态，一并补齐。
